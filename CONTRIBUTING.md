# Contributing to Azokle Calculator

First off, thank you for considering contributing to **Azokle Calculator**! It's people like you that make Azokle Calculator such a great tool for everyone.

Azokle Calculator is developed by the **Azokle Software** division of **Azokle Private Limited**. We are driven by our tagline: *Innovate. Integrate. Inspire.* and we strictly adhere to a **Privacy-First Technology** focus.

## Code of Conduct

By participating in this project, you are expected to uphold our [Code of Conduct](CODE_OF_CONDUCT.md).

## How Can I Contribute?

### Reporting Bugs

If you find a bug, please create an issue on our [GitHub Issues page](https://github.com/azoklesoftware/azokle-calculator-android/issues). 
- Please include steps to reproduce the issue.
- Mention your Android version and device model.
- Attach any relevant screenshots or logs.

### Suggesting Enhancements

We are always looking for new ideas! To suggest an enhancement:
- Check if your idea has already been suggested or implemented in the issue tracker.
- Clearly describe the feature and how it benefits the users.
- Remember our core focus: **Privacy-First**. Features that require tracking, telemetry, or unnecessary internet permissions will be rejected.

### Your First Code Contribution

If you are new to the project, look for issues labeled `good first issue` or `help wanted`.

1. **Fork the repository**: Click the Fork button on the top right corner of the GitHub page.
2. **Clone the repository**: `git clone https://github.com/YOUR_USERNAME/azokle-calculator.git`
3. **Create a branch**: `git checkout -b feature/your-feature-name`
4. **Make your changes**: Write your code, ensuring it follows the existing Kotlin and Compose conventions.
5. **Commit your changes**: `git commit -m "Add some feature"`
6. **Push to the branch**: `git push origin feature/your-feature-name`
7. **Open a Pull Request**: Submit your PR against the `main` or `master` branch.

### Pull Request Process

1. Ensure any install or build dependencies are removed before the end of the layer when doing a build.
2. Update the README.md with details of changes to the interface, this includes new environment variables, exposed ports, useful file locations and container parameters.
3. You may merge the Pull Request in once you have the sign-off of at least one core developer, or if you do not have permission to do that, you may request the reviewer to merge it for you.
